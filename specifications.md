"Deepfake Detection" 명세서

1. 모델의 존재 이유 (Context)
   목적: 화상통화 중 실시간으로 유입되는 공격자의 딥페이크 영상을 차단하거나 경고하기 위함.
   특성: 단일 이미지가 아닌 20프레임의 '흐름(Temporal features)'을 보고 판별하는 시계열 모델. 즉, 찰나의 이미지가 아닌 '움직임의 어색함'을 잡아내는 것이 핵심.

2. 탐지 및 신뢰도 산출 로직 (Scoring Strategy)
   에이전트에게 단순 확률(0~1)만 출력하게 하지 말고, 누적 데이터 기반의 신뢰도를 계산하도록 시켜야 합니다.
   개별 점수: 모델이 뱉는 $1 \times 1$ 값 (예: 0.85 = 85% 확률로 딥페이크).
   신뢰도 산출 방식 (Moving Average): \* 최근 5번의 추론 결과(총 100프레임 분량)를 평균 내어 최종 신뢰도를 산출.이유: 일시적인 네트워크 노이즈나 조명 변화로 인한 오판을 방지하기 위함.
   로그 기록: 각 추론 시점의 [프레임 구간 / 개별 점수 / 누적 평균 점수 / 처리 속도]를 CSV 형태나 구조화된 로그로 남겨서 사후 검증 가능하게 함.

3. 시스템 아키텍처 (Structure)
   Producer: 카메라 스트림에서 Apple Vision으로 얼굴을 찾아 큐(Queue)에 넣음.Buffer: 20개가 차면 Isolate(Worker)로 데이터를 넘김.Consumer: Isolate에서 모델을 돌리고 결과를 메인 UI 스레드로 보고.Logger: 결과를 메모리에서 해제하기 전, 해당 프레임의 특징값이나 점수를 로컬 DB나 로그 파일에 기록.
